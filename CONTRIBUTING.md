# Contributing to Rootage Repositories

## FR - Portee

Nous acceptons les contributions externes sur les repositories publics de l'organisation.

## EN - Scope

We accept external contributions on public repositories across the organization.

## FR - Comment contribuer

1. Ouvrir une issue pour discuter du besoin (bug, proposition, enhancement)
2. Forker le repository concerne
3. Creer une branche explicite (`feature/...`, `fix/...`, `docs/...`)
4. Faire des commits clairs et atomiques
5. Ouvrir une Pull Request avec contexte, impact et tests

## EN - How to contribute

1. Open an issue to discuss the need (bug, proposal, enhancement)
2. Fork the relevant repository
3. Create a clear branch name (`feature/...`, `fix/...`, `docs/...`)
4. Write clear and atomic commits
5. Open a Pull Request with context, impact, and tests

## FR - Bonnes pratiques PR

- Decrire le probleme et la solution
- Ajouter une strategie de verification (tests, captures, etapes)
- Garder les PR petites et reviewables
- Respecter la convention de code du projet cible

## EN - PR best practices

- Describe the problem and the proposed solution
- Add a verification strategy (tests, screenshots, steps)
- Keep PRs small and reviewable
- Follow the code style of the target project

## FR/EN - Code of Conduct

En contribuant, vous acceptez notre Code of Conduct: [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)

By contributing, you agree to follow our Code of Conduct: [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)
